# Handoff

Use this when the issue has a PR or a clear blocker.

## Ready Handoff

Include:

- Issue link.
- PR link.
- Summary of changes.
- Validation evidence.
- Feedback sweep status.
- Known risks.

Keep it concise. Put durable detail in the workpad and PR body.

## Blocked Handoff

Include:

- What is missing.
- Why it blocks completion.
- What exact action unblocks it.
- What work has already been completed.

Do not label blocked work as ready.
